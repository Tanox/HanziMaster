Part 04

交互标准

一致的用户体验来自一致的交互规则。

交互模式库
01

Hover-First

所有可交互元素应有明确的悬停状态（颜色变化、轻微上浮、边框高亮）。

02

Keyboard First

主要操作应支持键盘快捷键（Enter 确认，Esc 关闭，Tab 切换焦点）。

03

Touch Friendly

触摸目标最小 44×44 px，按钮内部保留充足间距。

04

Optimal Feedback

用户操作应在 100ms 内有视觉反馈（缩放、颜色、位移）。

错误处理规范
!
表单验证错误
显示字段级错误 + 顶部总结；第一次错误字段自动 focus；使用 shake 动画吸引注意。
⚠
网络错误
离线时保留已缓存的学习内容；显示离线徽章；重连后自动同步。
ℹ
空状态
所有列表应有明确的空状态插图 + 说明文案 + 行动按钮，避免用户看到空白页面。
组件使用规则

单一职责

每个组件只做一件事。复合组件通过组合基础组件形成。

可访问性

所有组件必须可键盘操作；提供语义化 HTML 和 ARIA 属性。

主题支持

颜色通过 CSS 变量表达，自动支持浅色与深色模式。

响应式

移动端使用纵向堆叠，桌面端使用横向布局；断点 ≥ 768。
