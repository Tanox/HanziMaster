# 中文字笔画顺序教学内容规范

## 1. 笔画演示动画标准

### 1.1 动画速度
- **默认速度**：每个笔画的绘制时间应控制在 0.8秒 至 1.2秒 之间，以确保用户能够清晰观察运笔轨迹。
- **连贯性**：笔画之间应有 0.2秒 的停顿，模拟真实书写时的提笔动作。
- **可调节性**：提供 0.5x (慢速)、1.0x (正常)、1.5x (快速) 三档速度选项，满足不同学习阶段的需求。

### 1.2 动画效果
- **轨迹显示**：笔画应沿着正确的书写路径动态填充，使用深色（如黑色或深灰色）显示当前笔画。
- **未书写部分**：未书写的笔画应以浅灰色轮廓或半透明底色显示，作为整体结构的参考。
- **当前笔画高亮**：正在演示的笔画应有明显的高亮效果（如颜色加深或边缘发光），引导用户视线。
- **起笔/收笔提示**：在笔画的起点显示实心圆点，终点显示箭头或空心圆点，明确运笔方向。

### 1.3 辅助视觉元素
- **田字格/米字格**：背景应始终显示标准的田字格或米字格，辅助用户理解汉字的间架结构。
- **笔顺编号**：在每个笔画的起笔位置旁显示小号数字，标示该笔画的顺序。

## 2. 用户输入笔画校验规则

### 2.1 轨迹匹配
- **容差范围**：用户的书写轨迹必须在标准笔画路径的一定宽度范围内（例如标准路径宽度的 3-4 倍）。
- **方向检测**：必须检测书写方向。即使用户的轨迹在路径范围内，如果方向相反（如从下往上写竖），也应判定为错误。
- **完整度**：用户必须完成笔画的 80% 以上长度才算有效输入。

### 2.2 顺序校验
- **严格顺序**：用户必须严格按照正确的笔顺进行书写。如果用户尝试先写后续的笔画，系统应忽略该输入或给予提示。
- **连笔处理**：在初级教学模式下，不鼓励连笔，系统应要求用户每次提笔后重新定位起笔点。

### 2.3 错误判定
- **出界**：笔画严重偏离标准路径（超过容差范围）。
- **逆锋**：书写方向与标准相反。
- **笔画错误**：尝试书写错误的笔画（如在写“横”的时候写了“竖”）。

## 3. 练习反馈机制

### 3.1 实时反馈
- **正确提示**：当用户正确完成一笔时，该笔画应自动吸附到标准位置，并播放清脆的“正确”音效，同时笔画颜色变为标准色（如黑色）。
- **错误提示**：当用户书写错误（出界、逆锋、笔顺错）时，轨迹应显示为红色并带有抖动效果，播放低沉的“错误”音效，随后轨迹消失，提示用户重试。
- **引导提示**：如果用户连续错误 3 次，系统应自动播放当前笔画的演示动画，或高亮显示正确的起笔位置和轨迹。

### 3.2 练习结算
- **完成动画**：当用户写完所有笔画后，汉字应有一个整体的放大或闪光特效，表示练习完成。
- **评分标准**：
  - **准确度 (Accuracy)**：根据用户轨迹与标准路径的重合度打分（0-100分）。
  - **流畅度 (Fluency)**：根据书写速度的均匀性和停顿次数打分。
  - **综合评分**：结合准确度、流畅度和错误次数，给出一个总分或星级（1-3星）。
- **结果展示**：
  - 显示综合评分和星级。
  - 标记出写得最差的笔画，建议用户重点练习。
  - 提供“重写”和“下一个”按钮。
