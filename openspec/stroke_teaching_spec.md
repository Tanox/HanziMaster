# 中文字笔画顺序教学内容规范 (v1.0.0)

## 1. 笔画演示动画标准

### 1.1 动画速度与节奏
- **基础速度**：每个笔画的绘制时长应控制在 0.8秒 至 1.2秒 之间，确保用户能看清起笔、行笔和收笔的细节。
- **笔画间隔**：每个笔画完成后，应有 0.2秒 至 0.4秒 的停顿，以便用户消化当前笔画的形态。
- **整体节奏**：动画播放应流畅连贯，避免突兀的加速或减速。对于复杂汉字（超过10画），可提供“分步播放”功能。

### 1.2 视觉效果
- **笔画颜色**：
  - **当前笔画**：使用高对比度颜色（如深绿色 #10B981 或 鲜红色 #EF4444）高亮显示。
  - **已完成笔画**：使用深灰色（如 #374151）或黑色显示，保持清晰但不抢眼。
  - **未书写笔画**：使用浅灰色（如 #E5E7EB）轮廓或虚线显示，作为整体结构的参考。
- **笔锋表现**：动画应模拟真实毛笔或硬笔的书写轨迹，体现出起笔的顿挫和收笔的回锋（如悬针竖、垂露竖的区别）。
- **动态引导**：在笔画起始点可出现光标或圆点引导，沿书写路径移动，辅助用户理解运笔方向。

### 1.3 辅助信息
- **笔画名称**：在演示当前笔画时，同步显示或朗读该笔画的名称（如“横”、“竖”、“撇”、“捺”）。
- **笔顺编号**：在笔画旁标记数字序号，明确书写顺序。

## 2. 用户输入笔画校验规则

### 2.1 轨迹匹配度
- **判定标准**：用户书写的轨迹需在标准笔画路径的有效误差范围内（通常为笔画宽度的 1.5倍 至 2倍）。
- **起止点校验**：
  - **起笔点**：用户落笔点需在标准起笔点的允许半径内（如 30px）。
  - **收笔点**：用户抬笔点需在标准收笔点的允许半径内。
  - **方向校验**：必须严格遵循笔画的书写方向（如“从左到右”、“从上到下”），逆向书写即使轨迹重合也判为错误。

### 2.2 容错机制
- **连笔处理**：对于行书或快写模式，允许一定程度的连笔，但在基础教学模式下应严格要求一笔一画。
- **抖动修正**：系统应对用户的书写轨迹进行平滑处理，消除因手指或触控笔抖动产生的微小锯齿，但不应过度修正导致笔画变形。

### 2.3 实时反馈
- **正确反馈**：书写正确时，笔画自动吸附到标准位置，并伴随清脆的提示音或视觉高亮。
- **错误反馈**：
  - **轨迹偏离**：笔画颜色变红并抖动，提示用户重写。
  - **笔顺错误**：提示“笔顺错误”，并演示正确笔顺。
  - **方向错误**：提示“方向错误”，显示正确的运笔箭头。

## 3. 练习反馈与评价机制

### 3.1 评分维度
- **准确度 (Accuracy)**：用户轨迹与标准模板的重合程度（权重 60%）。
- **流畅度 (Fluency)**：书写过程中的速度均匀性和连贯性（权重 20%）。
- **完整度 (Completeness)**：是否完成了汉字的所有笔画（权重 20%）。

### 3.2 评分等级
- **神品 (Exquisite)**：综合得分 ≥ 95分。笔画精准，结构完美。
- **妙品 (Masterful)**：90分 ≤ 综合得分 < 95分。笔画流畅，偶有微小偏差。
- **能品 (Proficient)**：80分 ≤ 综合得分 < 90分。结构正确，但笔画细节有待提升。
- **须努力 (Needs Improvement)**：综合得分 < 80分。存在明显的笔顺或结构错误。

### 3.3 结果展示
- **总分显示**：练习结束后，醒目展示本次练习的总分。
- **错误回顾**：高亮显示书写最差的 1-3 个笔画，并提供针对性的改进建议（如“横画不够平直”、“撇画角度过大”）。
- **对比功能**：将用户的字迹与标准字帖重叠显示，通过透明度调节，让用户直观看到差距。
- **鼓励机制**：根据得分给予相应的勋章或星星奖励，激发用户的学习兴趣。
