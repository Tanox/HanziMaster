# 03. UI/UX 设计规范

## 1. 设计哲学
*   **学术与现代并重**: 汉字展示使用衬线体 (Noto Serif SC) 传达传统感，UI 界面使用无衬线体 (Inter) 传达现代感。
*   **焦点驱动**: 汉字动画是舞台中心。所有控制控件都是次要的。
*   **响应式**: 布局从单列 (移动端) 平滑过渡到双列 (桌面端)。

## 2. 设计变量 (Design Tokens - Tailwind)

### 2.1 调色板
*   **品牌色 (Brand)**: `Vermilion` (朱砂红)。
    *   HEX: `#cf352e`
    *   用途: App 图标、印章效果、笔画临摹的主色调。
*   **主色 (Primary)**: `Teal-600` (亮色) / `Teal-400` (暗色)。用于按钮、链接和高亮状态。
*   **背景色**: `Slate-50` / `Paper` (`#fdfbf7` 米纸色) (亮色) / `Slate-900` (暗色)。
*   **反馈色**:
    *   成功: 隐式反馈（自动进入下一笔）。
    *   错误: `Red-500` (临摹错误时闪烁红色笔画)。

### 2.2 排版
*   **UI 字体**: `Inter`, system-ui.
*   **汉字字体**: `Noto Serif SC`. 必须使用 Serif 字体以匹配 SVG 笔画的笔锋风格。

## 3. 组件规范

### 3.1 笔顺播放器 (StrokeViewer)
*   **容器**: 保持 1:1 宽高比，最大宽度 400px。
*   **背景**: 支持切换“米字格”、“田字格”或无背景。
*   **技术实现**:
    *   **演示模式**: 使用 SVG `<mask >` 和 `stroke-dasharray` 动画路径。
    *   **临摹模式**: 使用 HTML5 `<canvas>` 覆盖层捕获 `pointer` 事件。
    *   **坐标系转换**: 需要处理 SVG 坐标 (1024x1024, Y轴向下) 与 Hanzi Writer 数据 (Y轴向上) 之间的转换。

### 3.2 解析面板 (AnalysisPanel)
*   **布局**: Bento Grid (便当盒) 风格的自适应网格。
*   **状态**:
    *   **Loading**: 显示与文本行高匹配的骨架屏 (Skeleton)。
    *   **Fallback**: 当 AI 不可用时，显示精简版信息，隐藏词源和记忆卡片。

### 3.3 历史与推荐 (History & Suggestions)
*   **随机推荐**: 位于底部，显示 3-6 个大号汉字卡片，支持悬停动效。
*   **历史面板**: 显示最近查看的汉字，支持一键清空。

## 4. 无障碍性 (A11y)
*   **主题**: 遵循系统 `prefers-color-scheme` 设置，支持手动切换。
*   **对比度**: 文本颜色必须符合 WCAG AA 标准。
*   **键盘导航**: `SearchInput` -> `Controls` -> `AnalysisPanel` 的 Tab 顺序应当逻辑连贯。

## 5. 动画
*   **微交互**: 按钮悬停效果（轻微放大 `scale-105`）。
*   **过渡**: 
    *   AI 解析内容加载完成后使用 `animate-fade-in` 淡入。
    *   临摹成功时显示“优”字印章动画 (`animate-bounce`).
