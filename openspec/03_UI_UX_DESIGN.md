# 03. UI/UX 设计规范

**版本**: v0.7.1
**状态**: 现行规范

## 1. 设计哲学：纸墨精神 (Paper & Ink)
HanziMaster 的核心隐喻是**“宣纸与朱砂”**。界面应传达出传统书房的静谧与专注，而非科技产品的冰冷。所有设计决策都应服务于沉浸式、无干扰的学习体验。

## 2. 主题与色彩规范

### 2.1 物理纹理 (Physical Texture)
*   **宣纸底纹**: 应用背景必须叠加 SVG 噪点滤镜 (`bg-texture-paper`)，模拟纸张的纤维感，增加视觉深度。
*   **光照模式**: 强制 **Light Mode** 为主要教学环境，因为它最能还原真实的纸墨体验。Dark Mode 仅作为夜间辅助，且同样保留纹理以避免纯黑的死板。

### 2.2 调色板 (Color Palette)
*   **背景 (Paper)**: `#fdfbf7` (暖白色，模拟宣纸)
*   **墨色 (Ink)**: `#0f172a` (Slate 900，用于正文和已完成的笔画)
*   **朱砂 (Vermilion)**: `#bc221b`
    *   **用途**: **教学高亮** (当前正在书写的笔画)、品牌标识、印章、关键行动点。
*   **翡翠 (Jade/Teal)**: `#0d9488` (Teal 600)
    *   **用途**: **成功反馈**、主要操作按钮、选中状态。
*   **胭脂 (Error/Red)**: `#ef4444`
    *   **用途**: **错误反馈** (例如错误的临摹轨迹)。

## 3. 教学交互规范 (Teaching Interaction)

### 3.1 笔顺演示 (Viewing Mode)
*   **未书写部分**: 浅灰色 (`slate-200`) 轮廓。
*   **流体动画**: 笔画必须呈现出“书写中”的动态长度变化 (`stroke-dashoffset` 动画)，模拟毛笔行进过程。

### 3.2 临摹练习 (Practice Mode)
*   **画笔**: 使用类似毛笔的笔触效果 (Canvas `lineCap='round'`)，粗细适中。
*   **反馈机制**:
    *   **Ghosting Hint**: 连续错误 3 次后，显示淡红色的半透明笔画轮廓，并在笔画中心显示脉冲动画引导。
    *   **判定成功**: 用户笔迹闪烁一次翡翠色，然后变为黑色定格，成为背景的一部分。
    *   **判定失败**: 用户笔迹变为胭脂红色并停留 0.5秒，随后消失，提示用户重写当前笔画。

## 4. 字体排印 (Typography)
*   **汉字 (Hanzi)**: `Noto Serif SC` (思源宋体) - **必须**使用衬线体以展示汉字的笔锋美感。
*   **西文/UI (Sans)**: `Inter` - 使用现代无衬线体，保持界面清晰易读，不干扰汉字主体。

## 5. 布局与响应式 (Layout & Responsive)
*   **移动端优先**: 核心书写区域 (Canvas) 在移动端必须占据屏幕宽度的 90% 以上。
*   **触摸区域 (Touch Target)**: 所有操作按钮（播放、重置、发音等）必须满足至少 44x44px 的触摸热区，以符合 WCAG 无障碍标准。
