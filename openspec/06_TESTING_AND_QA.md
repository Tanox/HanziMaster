# 06. 测试与质量保证 (QA)

**版本**: v0.7.1
**状态**: 现行规范

## 1. 测试总览
HanziMaster 的测试策略侧重于**核心教学逻辑的准确性**和**离线环境的健壮性**。

## 2. 核心测试用例 (Core Test Cases)

### 2.1 教学逻辑测试 (Pedagogy Testing)
*   **CASE-P-01: 虚影救生圈 (Ghosting Lifeline)**
    *   **步骤**: 在练习模式下，对同一笔画故意连续写错 3 次（起点、方向或形状错误均可）。
    *   **预期**: 第 3 次错误反馈后，当前笔画的半透明轮廓应自动显现，并伴有轻微脉冲动画。
*   **CASE-P-02: 严格模式判定 - 方向错误**
    *   **步骤**: 尝试反向书写一个笔画（如“一”从右向左写）。
    *   **预期**: 系统应立即判定为错误，显示红色轨迹，并增加错误计数。
*   **CASE-P-03: 严格模式判定 - 起点错误**
    *   **步骤**: 在距离正确起点很远的位置开始书写。
    *   **预期**: 落笔即触发错误反馈，无法开始绘制轨迹。

### 2.2 离线韧性与数据审计测试 (Offline Resilience)
*   **CASE-O-01: 笔顺库同步**
    *   **步骤**: 清空浏览器缓存 -> 进入“设置 -> 数据与同步” -> 点击“下载完整笔顺库”。
    *   **预期**: 进度条应平滑推进至 100%，并在完成后弹出成功提示。中途断网再联网，应能继续下载。
*   **CASE-O-02: 完全离线模式验证**
    *   **步骤**: 完成数据同步 -> 开启飞行模式 -> 完全关闭并重启应用。
    *   **预期**: 应用能正常启动。搜索常用字（如“爱”），应能加载笔顺动画和本地释义。AI 分析部分应明确提示“需要联网”。

### 2.3 兼容性与性能 (Compatibility & Performance)
*   **浏览器**: 必须在最新版的 Chrome, Safari, Firefox 上通过核心功能测试。
*   **移动端**:
    *   **触控延迟 (Input Latency)**: 书写延迟必须低于 16ms (确保 60fps 流畅度)。
    *   **iOS Safari**: 验证 Canvas 区域已设置 `touch-action: none`，以阻止页面在书写时的橡皮筋回弹效果。

### 2.4 国际化验证 (I18N Audit)
*   **完整性检查**: 确保所有语言文件 (`app/locales/*.ts`) 均包含最新功能的键值（如 `downloadLexicon`, `pinyinCoverage`）。
*   **UI 溢出**: 在德语 (de) 和俄语 (ru) 环境下，检查设置面板的按钮文字是否存在溢出或不当换行。
