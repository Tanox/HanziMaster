# Git 提交模板

## 提交消息规范
基于 Angular 提交规范：

```
<类型>: <描述>

[可选正文]

[可选页脚]
```

## 类型 (Type)
- `feat`: 新功能
- `fix`: 缺陷修复
- `docs`: 文档变更
- `style`: 代码格式变更（不影响功能）
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试添加/修改
- `build`: 构建系统变更
- `ci`: CI/CD 配置变更
- `chore`: 其他杂项变更

## 规范
- 标题限制在 50 字符以内
- 首字母小写
- 使用祈使语气
- 不使用句号结尾

## 示例
```
feat: 在学习页面实现字符选择功能
fix: 解决主题切换组件初始化问题
docs: 更新项目概述文档
refactor: 优化主题持久化逻辑
```
