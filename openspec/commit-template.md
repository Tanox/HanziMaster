# Git 提交模板

## 1. 提交消息规范

基于 Angular 提交规范，格式如下：

```
<类型>: <描述>

[可选正文]

[可选页脚]
```

## 2. 类型 (Type)

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 缺陷修复 |
| `docs` | 文档变更 |
| `style` | 代码格式变更（不影响功能） |
| `refactor` | 代码重构 |
| `perf` | 性能优化 |
| `test` | 测试添加/修改 |
| `build` | 构建系统变更 |
| `ci` | CI/CD 配置变更 |
| `chore` | 其他杂项变更 |

## 3. 格式规范

| 规范项 | 要求 |
|--------|------|
| 标题长度 | 限制在 50 字符以内 |
| 首字母 | 小写 |
| 语气 | 祈使语气 |
| 结尾 | 不使用句号 |

## 4. 示例

```bash
feat: 在学习页面实现字符选择功能
fix: 解决主题切换组件初始化问题
docs: 更新项目概述文档
refactor: 优化主题持久化逻辑
```

## 5. 提交模板文件

将以下内容保存为 `.gitmessage` 文件：

```
# <类型>: <描述> (限制在 50 字符以内)
# 
# [可选正文] - 详细描述更改内容
# 
# [可选页脚] - 引用 issue 或 PR 编号
# 
# 类型: feat, fix, docs, style, refactor, perf, test, build, ci, chore
```
