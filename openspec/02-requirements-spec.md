## 1. 概述

本文档详细描述 HanziMaster 汉字大师平台的功能需求和非功能需求。平台是一个 AI 驱动的中文字符学习 Web 应用，采用 Apple 设计风格（纯白背景、毛玻璃效果、emerald 绿色强调、圆角 24px、柔和阴影）。

---

## 2. 功能需求 (Functional Requirements)

### 2.1 FR-001: 首页展示

| 属性 | 描述 |
|-----|------|
| **需求编号** | FR-001 |
| **优先级** | 高 |
| **描述** | 首页展示应用介绍、核心功能卡片和主要入口按钮 |

**详细需求：**
- 展示 AI 驱动学习的状态徽章
- 展示主标题和副标题，使用渐变色文字效果
- 展示应用简介描述
- 提供「开始学习」和「探索字库」两个 CTA 按钮
- 展示 3 个功能特性卡片（AI 洞察、词源文化、自适应学习）
- 响应式布局：移动端单列，桌面端三列
- 卡片圆角 24px，带有柔和阴影

**验收标准：**
- [ ] 首页在所有设备尺寸下正确显示
- [ ] CTA 按钮点击可导航至对应页面
- [ ] 卡片悬停时有轻微上浮动画
- [ ] `prefers-reduced-motion` 时禁用悬停动画

### 2.4 FR-004: 主题切换系统

| 属性 | 描述 |
|-----|------|
| **需求编号** | FR-004 |
| **优先级** | 高 |
| **描述** | 支持深色/浅色/跟随系统三态主题切换 |

**详细需求：**
- 三态循环切换：浅色 → 深色 → 系统 → 浅色
- 每个状态对应 SVG 图标（太阳/月亮/显示器）
- 悬停显示工具提示（国际化文本）
- `prefers-reduced-motion` 时平滑过渡降级
- 主题偏好持久化到 localStorage（key: `hanzi-master-theme`）
- 毛玻璃效果在两种主题下均正确显示：
  - 浅色模式：`backdrop-blur(20px) + rgba(255,255,255,0.7)`
  - 深色模式：`backdrop-blur(20px) + rgba(15,23,42,0.7)`

**验收标准：**
- [ ] 点击切换按钮循环切换三个状态
- [ ] 主题切换后页面色彩即时更新
- [ ] 刷新页面后保持上次选择的主题
- [ ] 选择「系统」时跟随操作系统主题变化
- [ ] 焦点环为 emerald 绿色 `ring-2 ring-emerald-500`

### 2.5 FR-005: 语言切换系统

| 属性 | 描述 |
|-----|------|
| **需求编号** | FR-005 |
| **优先级** | 高 |
| **描述** | 支持 11 种语言切换，含完整 ARIA 无障碍支持 |

**详细需求：**
- 下拉菜单选择语言（11 种）
- 当前选中语言高亮显示
- 最大高度 `70vh` 限制，超出滚动
- 完整 ARIA 属性：`role="listbox"`, `aria-expanded`, `aria-haspopup`
- 键盘导航：Escape 关闭，Arrow Up/Down 移动，Enter 选择
- 点击外部区域自动关闭
- 语言偏好持久化到 localStorage（key: `hanzi-master-locale`）
- 同步更新 `document.documentElement.lang`

**支持语言：**
| 代码 | 语言 | 代码 | 语言 |
|-----|------|-----|------|
| `en` | 英语 | `fr` | 法语 |
| `zh-CN` | 简体中文 | `pt-BR` | 葡萄牙语(巴西) |
| `zh-TW` | 繁体中文 | `de` | 德语 |
| `es` | 西班牙语 | `ja` | 日语 |
| `ar` | 阿拉伯语 | `ko` | 韩语 |
| `ru` | 俄语 | | |

**验收标准：**
- [ ] 下拉菜单可正确展开/收起
- [ ] 所有 11 种语言可选，选中后页面文本即时切换
- [ ] 键盘可完全操作语言切换
- [ ] 屏幕阅读器可正确朗读菜单状态
- [ ] ARRT 触控目标 ≥ 44x44px

## 3. 非功能需求 (Non-Functional Requirements)

### 3.1 NFR-001: 设计风格一致性

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-001 |
| **类型** | 设计/可用性 |
| **描述** | 全平台遵循 Apple 设计风格 |

**规范要点：**
- **背景**：纯白 `#ffffff`（浅色模式）/ 深海军 `#0f172a`（深色模式）
- **强调色**：emerald 绿色 `#10b981`，hover 状态 `#059669`
- **圆角**：大卡片 `24px`，中卡片 `16px`，按钮 `12px`
- **阴影**：三层柔和阴影系统
  - 浮起阴影：`0 20px 60px -15px rgba(0,0,0,0.15)`
  - 卡片阴影：`0 8px 24px -8px rgba(0,0,0,0.08)`
  - 微阴影：`0 2px 8px -4px rgba(0,0,0,0.05)`
- **毛玻璃**：`backdrop-blur(20px)` + 半透明背景
- **字间距**：`tracking-tight` 用于标题，`tracking-normal` 用于正文
- **动画缓动**：`cubic-bezier(0.4, 0, 0.2, 1)`

---

### 3.2 NFR-002: 无障碍 (A11y)

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-002 |
| **类型** | 可访问性 |
| **描述** | 满足 WCAG 2.1 AA 级可访问性要求 |

**规范要点：**

**ARIA 属性：**
- 所有纯图标按钮必须有 `aria-label`
- 下拉菜单使用 `role="listbox"` + `role="option"` + `aria-selected`
- 对话框使用 `role="dialog"` + `aria-modal="true"` + `aria-labelledby`
- 状态切换按钮使用 `aria-pressed` 或 `aria-checked`
- 动态内容区域使用 `aria-live="polite"`

**键盘导航：**
- 所有交互元素必须可通过 Tab 键聚焦
- Tab 顺序与视觉顺序一致
- 交互元素必须有可见的焦点样式（`ring-2 ring-emerald-500`）
- Escape 键关闭所有弹出层
- 对话框打开时锁定焦点在对话框内

**Skip Link：**
- 页面顶部提供「跳转至主内容」链接（`<a href="#main-content">`）
- 默认视觉隐藏，Tab 聚焦时显示
- 位于 DOM 顺序第一位

**对比度：**
- 正文文本与背景对比度 ≥ 4.5:1
- 大号文本与背景对比度 ≥ 3:1
- 非文本交互元素焦点指示对比度 ≥ 3:1

**动效偏好：**
- 检测 `@media (prefers-reduced-motion: reduce)`
- 启用时禁用所有装饰性动画（仅保留必要的状态切换）
- 过渡时间减少为 0 或极短值

**触控目标：**
- 所有可点击/可交互元素最小尺寸 44x44px
- 元素间保持适当间距避免误触

---

### 3.3 NFR-003: 响应式设计

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-003 |
| **类型** | 可用性/适配 |
| **描述** | 完美适配从手机到桌面的全设备尺寸 |

**断点系统：**

| 断点名称 | 宽度范围 | 设备类型 | 布局策略 |
|---------|---------|---------|---------|
| `xs` | < 475px | 小手机 | 单列紧凑布局，字体略缩小 |
| `sm` | 475px - 768px | 手机 | 单列布局，底部导航栏 |
| `md` | 768px - 1024px | 平板 | 双列布局，顶部导航 + 抽屉菜单 |
| `lg` | 1024px - 1280px | 笔记本 | 三列布局，完整顶部导航 |
| `xl` | ≥ 1280px | 桌面 | 三列布局，最大内容宽度限制 |

**移动优先原则：**
- 样式以移动端为基准，使用 `md:` `lg:` 前缀扩展到更大设备
- 移动端底部导航栏替代桌面端顶部导航
- 触控目标在移动端保持 ≥ 44px 最小尺寸

---

### 3.4 NFR-004: 性能要求

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-004 |
| **类型** | 性能 |

**指标要求：**
- **首屏加载 (LCP)**：< 2.5 秒（移动 3G），< 1 秒（桌面宽带）
- **可交互时间 (TTI)**：< 3.5 秒
- **累积布局偏移 (CLS)**：< 0.1
- **总页面大小**：< 500KB（未压缩）
- **核心 Web 指标**：全部达到「良好」等级

**优化策略：**
- 使用 Next.js Server Components 减少客户端 JS
- 字体预加载和 `font-display: swap`
- Tailwind CSS 4.0 按需编译，零运行时开销
- `prefers-reduced-motion` 时减少动画计算

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-005 |
| **类型** | 兼容性 |

**支持浏览器：**
| 浏览器 | 最低版本 | 说明 |
|--------|---------|------|
| Chrome | 100+ | 全功能支持 |

| Safari | 15.4+ | 需要 CSS backdrop-filter 支持 |
| Edge | 100+ | 全功能支持 |

### 3.6 NFR-006: 国际化 (i18n)

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-006 |
| **类型** | 国际化 |

### 3.7 NFR-007: 安全性

| 属性 | 描述 |
|-----|------|
| **需求编号** | NFR-007 |
| **类型** | 安全 |

**要求：**
- API Key 仅存储在环境变量中，从不暴露到客户端
- 用户输入（AI 功能部分）进行适当转义和长度限制
