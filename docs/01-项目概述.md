# 项目概述

**HanziMaster (汉字大师)** - 数字化书法教室

---

## 1. 项目背景

### 1.1 项目简介
HanziMaster 是一个融合传统书法美学与现代 AI 智慧的**沉浸式汉字教学平台**。它不仅仅是一个字典，更是一个完整的**教学终端**，通过 **Scaffolding 教学法**（Watch → Trace → Write）帮助用户掌握汉字书写。

### 1.2 项目愿景
融合传统书法美学与 AI 智慧，打造全球领先的**沉浸式汉字教学平台**。我们不仅教如何"写"，更致力于传承汉字背后的文化温度与美学价值。

### 1.3 核心理念
- **离线优先 (Offline-First)**: 任何网络请求都必须包含 Fallback 机制
- **AI 增强 (AI-Enhanced)**: 利用 Gemini AI 提供深度的汉字解析
- **零知识 (Zero-Knowledge)**: 所有用户数据仅存储于浏览器本地

### 4.2 Hooks 分层架构

为避免 `useAppController` 成为上帝对象，状态管理严格按下述原则抽离：

| Hook | 职责 |
|------|------|
| **useAppController** (协调者) | 应用的"大脑"。唯一持有 `activeChar` 与 `activeTerm`。负责 URL 解析、全局 Modal 状态、以及协调各子 Hook 间的交互。 |
| **useContentFetcher** (数据源) | 负责获取与缓存 `HanziData` (笔顺)、`CharacterAnalysis` (AI 解析) 和 `IdiomAnalysis`。 |
| **useInteractionState** (交互态) | 负责播放、重置、模式切换 (Watch/Practice) 等瞬时 UI 状态。 |
| **useUserProgress** (用户进度) | 独立管理用户的学习历史 (`HistoryItem[]`) 和已学清单 (`learnedItems: string[]`)，并与 `LocalStorage` 通信。 |
| **useDataSync** (数据同步) | 封装所有离线数据（笔顺库、词典）的下载、缓存与审计逻辑。 |

## 5. 核心设计原则

### 5.1 UI/UX 设计原则
- **色彩**: 遵循 "Vermilion (朱砂)" + "Slate (松烟)" 配色体系
- **字体**: 汉字必须使用 `Noto Serif SC`，UI 文本使用 `Inter`
- **响应式**: 必须优先适配移动端触摸操作 (44px+ 点击区域)
- **交互反馈**: 严禁使用浏览器原生 `alert()` 或 `confirm()`，必须使用全局 `Toast` 组件

### 5.2 编码规范
- **组件**: 函数式组件，使用 `React.FC<Props>` 类型定义
- **Props**: 必须显式定义 Interface，禁止使用 `any`
- **Hooks**: 遵循 Hooks 规则，复杂的 `useEffect` 必须在注释中说明依赖逻辑
- **异步**: 优先使用 Async/Await，所有 API 调用必须包裹在 `try/catch` 中
- **TypeScript**: 开启 `strict: true`，严禁 Implicit Any

### 5.3 文件规范
- **文件编码**: UTF-8, CRLF 行尾
- **文件头部注释**: 每个 `.ts` / `.tsx` 文件首行必须包含版本注释
- **导入**: 优先使用相对路径引入本地模块
