# 安装部署指南

---

## 1. 环境依赖清单

### 1.1 开发环境要求

| 依赖 | 最低版本 | 推荐版本 | 说明 |
|------|----------|----------|------|
| **Node.js** | 18.0.0 | 20.x | JavaScript 运行时 |
| **npm** | 9.0.0 | 10.x | 包管理器 |
| **Git** | 2.0.0 | 最新 | 版本控制 |
| **现代浏览器** | - | Chrome/Edge/Firefox | 开发测试 |

### 1.2 可选工具

- **bun**: 1.0+ (作为 npm 的替代方案)
- **VS Code**: 推荐 IDE
- **GitHub CLI**: 用于 GitHub 操作

## 2. 本地开发环境配置

### 2.1 克隆仓库

```bash
# 1. 克隆项目
git clone https://github.com/sutchan/hanzimaster.git
cd hanzimaster

# 2. 查看版本
git log -1
```

### 2.2 安装依赖

```bash
# 使用 npm 安装（推荐）
npm install

# 或使用 bun（更快）
bun install
```

**说明**: `npm install` 会自动触发 `postinstall` 钩子，执行数据同步脚本，将汉字笔顺数据从 `node_modules/hanzi-writer-data/` 复制到 `public/hanzi-data/` 目录。

### 2.3 环境变量配置

1. 复制环境变量示例文件：
```bash
# 如果存在 .env.example
cp .env.example .env
```

2. 编辑 `.env` 文件，配置 API Key（可选）：
```env
# Google Gemini API Key（开发用，非必须）
# App 支持用户端输入 Key，所以此配置可选
VITE_API_KEY=your_gemini_api_key_here
```

**注意**: 生产环境部署时，建议限制 API Key 的 Referrer 仅为官方域名。

### 2.4 启动开发服务器

```bash
# 启动 Next.js 开发服务器
npm run dev

# 或使用 bun
bun dev
```

访问: http://localhost:3000

### 2.5 手动数据同步

如果发现汉字无法加载，可以手动运行数据拷贝脚本：

```bash
npm run copy-data
```

## 3. 生产环境构建

### 3.1 构建应用

```bash
# 1. 执行构建（包括数据同步）
npm run build

# 2. 本地预览生产构建
npm run start
```

访问: http://localhost:3000

### 3.2 构建产物说明

构建后，项目结构如下：
```
.next/                # Next.js 构建输出
public/               # 静态资源
├── hanzi-data/      # 汉字笔顺数据（9000+ JSON 文件）
├── favicon.svg
├── icon.svg
└── ...
```

## 4. 多环境部署方案

### 4.1 部署到 Vercel（推荐）

#### 自动部署
1. 连接 GitHub 仓库到 Vercel
2. 配置环境变量（如果需要）
3. 每次推送代码会自动部署

#### 手动部署
```bash
# 安装 Vercel CLI
npm i -g vercel

# 登录
vercel login

# 部署
vercel
```

### 4.2 部署到其他平台

#### Docker（自建）

创建 `Dockerfile`：
```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine
WORKDIR /app
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/public ./public
COPY --from=builder /app/package*.json ./
EXPOSE 3000
CMD ["npm", "start"]
```

#### Nginx 静态部署

对于纯静态部署，可以使用 `next export`（注意：某些 Next.js 功能需要 Node.js 服务器）。

### 4.3 环境变量配置

| 环境变量 | 说明 | 必需 |
|---------|------|------|
| `VITE_API_KEY` | Google Gemini API Key | 否 |

## 5. 验证方法

### 5.1 开发环境验证清单

- [ ] 开发服务器正常启动（`npm run dev`）
- [ ] 可以访问 http://localhost:3000
- [ ] 汉字笔顺可以正常显示和动画
- [ ] 搜索功能正常工作
- [ ] 设置面板可以正常打开
- [ ] 浏览器控制台无错误（检查 F12）

### 5.2 生产环境验证清单

- [ ] 构建成功（`npm run build` 无错误）
- [ ] 生产服务器正常启动（`npm run start`）
- [ ] PWA 功能正常（可以安装到桌面）
- [ ] 离线模式正常工作
- [ ] 响应式布局在移动端显示正常
- [ ] 无障碍功能可用

### 5.3 代码质量检查

```bash
# 运行 Lint
npm run lint

# TypeScript 类型检查（通过构建）
npm run build
```

### 5.4 功能测试步骤

1. **测试笔顺动画**
   - 搜索"永"字
   - 点击播放按钮
   - 确认笔顺动画流畅

2. **测试练习模式**
   - 切换到"练习"模式
   - 在画布上书写
   - 确认评分系统正常

3. **测试 AI 解析**
   - 确认可以看到字源、助记等信息
   - 测试发音功能

4. **测试离线模式**
   - 断网测试
   - 确认核心功能可用

## 6. 常见问题（快速参考）

### 6.1 汉字无法加载
**解决方案**: 手动运行数据同步脚本
```bash
npm run copy-data
```

### 6.2 构建失败
**解决方案**: 
```bash
# 清理 node_modules 和构建缓存
rm -rf node_modules .next
npm install
npm run build
```

### 6.3 TypeScript 错误
**解决方案**: 
```bash
# 检查 tsconfig.json 配置
# 确保所有依赖已正确安装
npm install
```
